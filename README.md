# Web Application Exercise

A little exercise to build a web application following an agile development process. See the [instructions](instructions.md) for more detail.

## Product vision statement

Our app helps students discover NYU study spaces through peer feedback, enabling smarter decisions about where to study and collaborate.

## User stories
### Primary User: NYU Students
1. As an NYU student, I want to browse all study locations near campus so that I don’t waste time searching for a quiet place to work.

2. As an NYU student, I want to read reviews about study spots so that I can choose a place that matches my noise tolerance and study preferences.

3. As an NYU student, I want to leave reviews and comments about study locations so that I can help other students find good places to study.

4. As an NYU student, I want to view a study spot’s noise level so that I can find a place that fits my specific study needs.

5. As an NYU student, I want to view a study spot’s seating availability so that I can find a place that fits my specific study needs.

6. As an NYU student, I want to view a study spot’s outlet access so that I can find a place that fits my specific study needs.

7. As an NYU student, I want to know places that can be used for group study.

8. As an NYU student, I want to check a place's hours so that I can make sure I am going to places I can sit at.

9. As an NYU student, I want to view spots based on location so that I can find places that are near me.

10. As an NYU student, I want to log into the app so that I can use it.

11. As an NYU student, I want to have a user in the app so that I can use it.

## Steps necessary to run the software

### 1. Copy environment file
cp .env.example .env

### 2. Build and run containers
docker-compose up --build -d
